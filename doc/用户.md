# 用户管理相关接口
需求：完成用户管理相关的接口，用于前端登录后进行用户信息渲染展示

于/routes创建user_c.py作该项目的控制器，prefix -> user
于/services创建user_service.py作该项目的业务代码

## 接口实现
于`/db/models`下创建UserSettingParam，参考以下前端ts代码完成对应model:
```typescript
interface UserSettingParam{
    user_uuid: string; // the unique sign about user
    username: string | null;
    email: string | null;
    phone: string | null;
    status: number
}
```

参考 `/routes/auth.py` 的写法规范，要统一响应参数

注意做好，user到UserSettingParam的映射哦！

### 获取用户设置信息
`GET /user/setting` RETURNING: `Result[UserSettingParam]`
参考 `/middleware/auth_handler`,尝试获取token中用户的唯一标识，查库并映射返回
注意：电话号码需要混淆后几位，**一定不要原样返回**

### 修改用户信息(只能修改用户名)
`POST /user/setting` QUERY_PARAM: `username(new)` RETURNING: `Result`
尝试通过token获取唯一标识更新用户名
